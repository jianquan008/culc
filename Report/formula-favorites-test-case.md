# 【计算器-公式收藏】测试用例

生成时间：2025-12-18 13:39:00

## 测试范围
基于需求文档，本次测试专注于公式收藏功能的核心特性：
- 公式保存和管理
- 分类系统
- 搜索功能  
- 快速使用功能
- UI质量检测

## 测试用例列表

### TC-001: 公式保存功能测试
**测试目标**: 验证用户能够成功保存计算表达式为收藏公式
**前置条件**: 计算器应用已启动，完成一个计算
**测试步骤**:
1. 在计算器中输入表达式 "2+3*4"
2. 点击等号计算得到结果
3. 查找并点击收藏按钮
4. 在弹出的保存对话框中输入公式名称 "基础混合运算"
5. 选择分类 "基础运算"
6. 点击保存按钮

**预期结果**:
- 显示收藏按钮
- 弹出保存对话框
- 成功保存公式
- 公式出现在收藏列表中

### TC-002: 分类管理功能测试
**测试目标**: 验证默认分类显示和分类切换功能
**测试步骤**:
1. 打开公式收藏面板
2. 检查默认分类是否显示（基础运算、科学计算、单位换算、财务计算、自定义）
3. 点击不同分类标签
4. 验证分类切换效果

**预期结果**:
- 显示5个默认分类
- 分类切换正常
- 当前选中分类高亮显示

### TC-003: 搜索功能测试
**测试目标**: 验证公式搜索功能的准确性和实时性
**前置条件**: 已保存至少3个不同名称的公式
**测试步骤**:
1. 在搜索框中输入关键词 "基础"
2. 观察搜索结果实时更新
3. 输入不存在的关键词 "xyz123"
4. 清空搜索框

**预期结果**:
- 实时显示匹配的公式
- 无结果时显示"未找到相关公式"提示
- 清空搜索框后恢复显示所有公式

### TC-004: 公式快速使用测试
**测试目标**: 验证单击填入和双击计算功能
**前置条件**: 已保存公式 "2+3*4"
**测试步骤**:
1. 单击保存的公式 "基础混合运算"
2. 检查计算器显示屏是否填入表达式
3. 双击同一公式
4. 检查是否直接显示计算结果

**预期结果**:
- 单击后表达式填入计算器
- 双击后直接显示计算结果 "14"
- 使用次数增加

### TC-005: UI质量检测测试
**测试目标**: 严格检测UI质量，确保无按钮压缩、变形等问题
**测试步骤**:
1. 截取公式收藏面板完整界面
2. 检查所有按钮形状和尺寸
3. 验证按钮间距和布局协调性
4. 检查文字显示完整性
5. 测试响应式布局效果

**预期结果**:
- 所有按钮保持正常形状（方形/圆形）
- 按钮高宽比合理（接近1:1）
- 按钮间距≥5px
- 文字显示完整无截断
- 布局协调美观

### TC-006: 公式编辑和删除测试
**测试目标**: 验证公式管理操作的完整性
**前置条件**: 已保存至少1个公式
**测试步骤**:
1. 右键点击已保存的公式
2. 选择编辑选项，修改公式名称
3. 保存修改
4. 右键点击公式，选择删除
5. 确认删除操作

**预期结果**:
- 编辑功能正常，名称修改成功
- 删除功能正常，公式从列表中移除
- 操作有确认提示
